# Manual de Usuario - Matriz de Planificación (MTEPS)

Este documento detalla las funcionalidades actualizadas y la nueva nomenclatura del sistema para el seguimiento institucional.

## 1. Rebranding Institucional
El sistema ha sido renombrado de **SISTEMA MeGAs** a **Matriz de Planificación**. Este cambio se refleja en:
- Título de la pestaña del navegador.
- Pantalla de inicio de sesión (Login).
- Barra lateral de navegación.
- Encabezados de los módulos principales.

## 2. Estructura del Menú
El menú de navegación ha sido estandarizado siguiendo el orden solicitado:
1. **Matriz de Planificación**: Gestión de ejes, resultados, estrategias, MeGAs, productos y tareas.
2. **Seguimiento**: Registro de avances físicos y carga de evidencias.
3. **Reporte**: Panel de control (Dashboard) con métricas clave y Top 5 de tareas.
4. **Usuarios**: Administración de funcionarios, directores y permisos.
5. **Auditoría**: Registro histórico de acciones realizadas en el sistema.
6. **Ayuda**: Documentación técnica y guías de usuario.

## 3. Modalidades de Registro (Matriz de Planificación)
El módulo principal ahora cuenta con un **Selector de Modo** en la parte superior derecha:

### A. Modo Jerárquico (Tradicional)
Siguen la estructura en cascada obligatoria:
- **Resultado** depende de un **Eje**.
- **MeGA** depende de una **Estrategia**.
- **Producto** depende de un **MeGA**.
- **Actividad** depende de un **Producto**.
- **Tarea** depende de una **Actividad**.

En este modo, las tablas muestran celdas combinadas para visualizar claramente a qué nivel superior pertenece cada elemento.

### B. Modo Aislado (No Relacionado)
Permite el registro de información de forma independiente sin depender de un elemento previo. 
- Ideal para tareas operativas que no están vinculadas directamente al POA o resultados estratégicos específicos.
- Al activar este modo, el sistema oculta los selectores de dependencias y simplifica el formulario de registro.
- Los registros creados en este modo aparecerán únicamente cuando el filtro esté en "Aislado".

## 4. Gestión de Tareas
- **Hitos vs. Temporal**: Al crear una tarea, se puede elegir entre seguimiento por fechas (Semanal/Diario) o por hitos (cantidad fija de logros sin fecha específica).
- **Responsables**: Cada tarea debe tener un funcionario asignado para su ejecución y un director para su supervisión.

## 5. Roles de Usuario
- **Administrador de Sistema**: Acceso total a configuración, auditoría y gestión de usuarios.
- **Director General**: Supervisión de las tareas y MeGAs asignados a su unidad organizacional.
- **Técnico / Operador**: Registro de avances y evidencias de sus tareas asignadas.
